# OmniMind Project Management

## Sprint Structure
- **Duration**: 2-week sprints
- **Daily Standup**: 9:30 AM EST
- **Sprint Planning**: Monday
- **Sprint Review**: Friday

## Development Workflow
1. Create feature branch: `git checkout -b feature/feature-name`
2. Commit changes with conventional commits
3. Create PR with template
4. Code review required
5. Merge to develop
6. Deploy to staging
7. QA testing
8. Merge to main

## Tech Stack Decisions
- **Frontend**: React + TypeScript + Vite
- **Backend**: Node.js + Express + PostgreSQL
- **AI**: OpenAI APIs + LangChain
- **Mobile**: React Native (for future)
- **Deployment**: Docker + AWS ECS

## Key Milestones
- **Week 1-2**: Basic Chrome extension for Gmail
- **Week 3-4**: Task extraction from emails
- **Week 5-6**: Basic web dashboard
- **Week 7-8**: Meeting transcription MVP
- **Week 9-10**: Notification system
- **Week 11-12**: Basic AI scheduling
